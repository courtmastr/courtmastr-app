How should player stats flow from Tournament → Global Player?

This is the most important data model decision. Everything else follows from it.

A

Dual-Write (App writes both)

When a match result is saved, the app writes to both the tournament record AND the global player stats simultaneously.

Pros

  • Fast reads — global player always up to date
  • No Cloud Functions needed
  • Simple to understand

Cons

  • Two writes every score update
  • If one write fails, data is out of sync
  • App code is responsible for keeping both in sync forever
B

Cloud Function Fanout (Recommended)

App writes match result to tournament only. A Firestore trigger automatically updates the global player stats in the background.

Pros

  • App only writes one place — no sync bugs
  • Global player is always derived from source of truth
  • Can recompute stats anytime by replaying matches

Cons

  • Slight delay (~1-2s) before global stats update
  • Requires a Cloud Function
C

Compute on Read (No sync)

Store nothing in the global player doc. When the player profile page loads, query all tournaments that player was in and aggregate stats on the fly.

Pros

  • Zero duplication — single source of truth
  • No sync logic at all
  • Stats always perfectly accurate

Cons

  • Profile page is slow to load (many Firestore reads)
  • Gets slower as player plays more tournaments
  • Expensive at scale
