Role & Access Model

Who owns what, and what they can see

ADMIN
WebApp Admin
✓ All organizations
✓ All tournaments
✓ All players (global)
✓ All users / roles
✓ Platform settings
✓ Reviews / reports
One or few per system. Manages the whole platform.
ORGANIZER
= Org Admin
✓ Own org profile (edit)
✓ Own org's tournaments (all)
✓ Players in own tournaments
✓ Registrations in own tournaments
✓ Global player list (read)
✗ Other orgs' data
✗ Platform settings
One organizer can manage multiple orgs. Each org they are a member of appears in their org switcher.
PLAYER
Global Identity
✓ Own global profile (edit)
✓ Own stats (read)
✓ Own registrations (read)
✓ Register for any tournament
✓ Public tournament data
✗ Other players' contact info
✗ Admin/org management
Player is linked by email. Belongs to no org explicitly — associated through tournament registrations.
How Organizer ↔ Org membership works
/organizations/{orgId}/members/{uid}
uid: string
role: "admin" | "organizer"
joinedAt: Timestamp
One document per user per org.
An organizer in 3 orgs has 3 membership docs.
User document (/users/{uid})
activeOrgId: string ← NEW
role: "admin" | "organizer" | ...
...existing fields
activeOrgId = which org the organizer is currently operating as. Changes when they switch org context.
