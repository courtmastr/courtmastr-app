# TNF 2026 Production Seed Run Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Safely seed the TNF 2026 tournament into production Firebase with full local verification first.

**Architecture:** Run the workbook-driven seed locally against emulators to verify data integrity, then execute against real production Firebase. The seed is idempotent — re-running it will detect the existing tournament by name and skip re-creation, only patching organizer access if needed.

**Tech Stack:** TypeScript, tsx, Firebase (Auth + Firestore), XLSX workbook parser, Firebase emulators

## Task 5: Handle Re-Runs (if needed)

**Purpose:** The seed is idempotent — re-running it is safe. This task documents expected behavior.

If you run `npm run seed:tnf2026:prod` again after a successful seed:

- The tournament is found by name → skipped (no duplicate created)
- Organizer IDs are merged in if they changed
- Output will show: `Found existing tournament: <id>`

**If you need a clean re-seed** (e.g., to fix bad data):
1. Manually delete `tournaments/<id>` and all subcollections in Firebase console
2. Optionally delete the global `players` and `playerEmailIndex` documents for TNF players
3. Re-run `npm run seed:tnf2026:prod`

> **Warning:** Deleting production Firestore data is irreversible. Only do this if the seed data is confirmed wrong and there are no real user writes on top of it.
