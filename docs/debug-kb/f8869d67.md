# KB Entry: Release verification fails when release-notes test reads live repo version

## Context
- Command: `npm run test:log -- --run`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: `release:deploy` temporarily bumped the package version to `2.0.0` before running release verification.

## Error Signature
```text
Error: Missing release notes for v2.0.0: /Users/ramc/Documents/Code/courtmaster-v2/docs/releases/v2.0.0.md
```

## Attempts

### Attempt 1
**Date**: 2026-04-01
**Change**: Reproduced the release state manually by bumping the package version to `2.0.0` and rerunning `npm run test -- --run`.

**Result**: ✅ Reproduced exactly one failing test: `tests/unit/release-notes-utils.test.ts`.

### Attempt 2
**Date**: 2026-04-01
**Change**: Updated `tests/unit/release-notes-utils.test.ts` to create its own temporary `package.json` and `docs/releases/v<version>.md` fixtures instead of reading the repo's live versioned files.

**Result**: ✅ The targeted release-notes test passed at `2.0.0`.

### Attempt 3
**Date**: 2026-04-01
**Change**: Reran the full Vitest suite while the repo version was still `2.0.0`.

**Result**: ✅ Full suite passed: `143` files, `630` tests.

## Root Cause
One release-notes unit test depended on the repository's current package version and real `docs/releases/` files, so it failed when `release:deploy` temporarily bumped the version before verification.

## Fix (final)
```bash
# Update the test to use temp fixtures rather than repo state
edit tests/unit/release-notes-utils.test.ts

# Verify in release-state conditions
npm version 2.0.0 --no-git-tag-version
npm run test -- --run tests/unit/release-notes-utils.test.ts
npm run test -- --run

# Restore repo version after verification
npm version 1.1.0 --no-git-tag-version
```
