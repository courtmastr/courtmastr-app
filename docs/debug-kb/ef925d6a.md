# KB Entry: TournamentSummaryView type mismatch for completed status checks

## Context
- Command: `npm run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Added reports summary view at `/tournaments/:tournamentId/reports/summary`.

## Error Signature
```
TS2345: Argument of type 'MatchStatus' is not assignable to parameter of type '"completed" | "walkover"'
```

## Attempts

### Attempt 1
**Date**: 2026-02-15
**Change**:
- Replaced literal tuple `['completed', 'walkover'] as const` with `Set<string>`.
- Switched checks from `.includes(match.status)` to `.has(match.status)` in `TournamentSummaryView.vue`.

**Result**:
✅ Build passed.

## Root Cause
TypeScript narrowed `.includes()` on a literal tuple to only accept `"completed" | "walkover"`, which conflicted with broader `MatchStatus`.

## Fix (final)
```bash
npm run build:log
```

In code, use `Set.has()` for membership checks against `MatchStatus`.
