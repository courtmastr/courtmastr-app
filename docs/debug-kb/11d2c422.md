# KB Entry: build:log fails in a clean worktree before root and functions dependencies are installed

## Context
- Command: `npm run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2/.worktrees/fix-match-control-compact-score`
- Recent changes: Added compact Match Control manual scoring access for `scheduled`/`ready` rows and extended the targeted unit coverage.

## Error Signature
```text
functions/src/*.ts: error TS2307: Cannot find module 'firebase-admin' or 'firebase-functions'
```

## Attempts

### Attempt 1
**Date**: 2026-04-17
**Change**: Re-ran `npm run build` and `npm run build:log` after restoring `.env.production` in the clean worktree.

**Result**:
❌ The Firebase env gate passed, but the TypeScript build failed immediately because the clean worktree had no installed root dependencies and no installed `functions/` dependencies.

### Attempt 2
**Date**: 2026-04-17
**Change**: Installed dependencies in both package roots, then re-ran the build gates.
```bash
npm install
npm --prefix functions install
npm run build
npm run build:log
```

**Result**:
✅ `npm run build` and `npm run build:log` both passed.

## Root Cause
The clean worktree started without `node_modules` in the repo root or `functions/`, so `vue-tsc -b` could not resolve the Firebase packages referenced by the functions TypeScript sources.

## Fix (final)
```bash
npm install
npm --prefix functions install
npm run build
npm run build:log
```

- [x] Logged build passes: `npm run build:log`
