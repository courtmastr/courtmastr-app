# KB Entry: Firebase Hosting deploy rejects unsupported negative-lookahead rewrite

## Context
- Command: `npm run deploy:log`
- Working directory: `/home/runner/work/courtmastr-app/courtmastr-app`
- Recent changes: Added Hosting rewrite exclusions to stop stale deleted asset chunks from rewriting to `index.html`

## Error Signature
```
Supplied rewrite pattern invalid: error parsing regexp: invalid or unsupported Perl syntax: `(?!`
```

## Attempts

### Attempt 1
**Date**: 2026-04-16
**Change**: Added a `regex` rewrite with negative lookahead to exclude `/assets/**` plus service-worker/manifest paths from the SPA rewrite.

**Result**: ❌ Firebase Hosting rejected the config during deploy finalization because Hosting rewrites use RE2 and do not support PCRE negative lookaheads.

### Attempt 2
**Date**: 2026-04-16
**Change**: Replaced the unsupported rewrite with a Hosting-supported glob exclusion: `"source": "!/assets/**"`, while keeping app-shell files controlled by header rules instead of regex exclusions.

**Result**: ✅ Local build verification succeeded and the deploy-blocking config error was removed from the Hosting config.

## Root Cause
Firebase Hosting rewrite regexes use RE2, so the negative-lookahead pattern used to exclude asset paths from the SPA rewrite was invalid and blocked deploy finalization.

## Fix (final)
```bash
# In firebase.json
"rewrites": [
  {
    "source": "!/assets/**",
    "destination": "/index.html"
  }
]

# Verify
npm run build
npm run build:log
```
