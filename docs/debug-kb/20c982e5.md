# KB Entry: lint:log fails when a hoisted Vitest runtime uses raw require('vue')

## Context
- Command: `npm run lint:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2/.worktrees/pool-standings-single-source`
- Recent changes: Added a `SmartBracketView` regression test while moving the `Pre-Elim Snapshot` view onto the shared pool standings helper.

## Error Signature
```
error  A `require()` style import is forbidden  @typescript-eslint/no-require-imports
```

## Attempts

### Attempt 1
**Date**: 2026-04-19
**Change**: Typed the hoisted `vue` import inside `tests/unit/SmartBracketView.test.ts` as `typeof import('vue')` and kept the one-line ESLint suppression scoped to the `vi.hoisted()` test harness that cannot use normal ESM imports.

**Result**:
✅ `npm run lint:log` passed. Existing repository warnings remained warnings only.

## Root Cause
The new Vitest harness used an untyped `require('vue')` inside `vi.hoisted()`, which is necessary for pre-import mock setup but violates the repo's ESLint rule unless the use is both typed and explicitly scoped.

## Fix (final)
```bash
# In vi.hoisted() test harnesses, type the Vue require and scope the lint suppression
const runtime = vi.hoisted(() => {
  // eslint-disable-next-line @typescript-eslint/no-require-imports
  const vue = require('vue') as typeof import('vue');
  return {
    categories: vue.ref([]),
  };
});

npm run lint:log
```
