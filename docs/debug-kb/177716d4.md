# KB Entry: GitHub OIDC deploy identity lacked Service Usage Consumer for Firebase deploy

## Context
- Command: `npm run release:deploy`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: merged the CI-driven deploy pipeline to `master` and first production release reached `firebase deploy` through GitHub OIDC.

## Error Signature
```text
Caller does not have required permission to use project. Grant the caller the roles/serviceusage.serviceUsageConsumer role, or a custom role with the serviceusage.services.use permission.
```

## Attempts

### Attempt 1
**Date**: 2026-04-02
**Change**: Pulled the failed GitHub Actions release log and isolated the failure to the dedicated deploy service account after `firebase deploy --project production` started.

**Result**: ✅ Confirmed the existing Terraform deploy IAM omitted `roles/serviceusage.serviceUsageConsumer`.

### Attempt 2
**Date**: 2026-04-02
**Change**: Added `roles/serviceusage.serviceUsageConsumer` to `infra/terraform/deploy/main.tf` and applied the deploy Terraform root with the existing production state.

**Result**: ✅ The deploy service account received the missing project role and the next release attempt passed the original Service Usage failure point.

## Root Cause
The Terraform-managed deploy service account had Firebase, storage, Cloud Functions, and Run permissions but was missing `roles/serviceusage.serviceUsageConsumer`, which Firebase CLI needs during production deploys.

## Fix (final)
```bash
cd /Users/ramc/Documents/Code/courtmaster-v2/infra/terraform/deploy
/Users/ramc/.local/bin/terraform-1.14.8 plan -var-file=terraform.tfvars
/Users/ramc/.local/bin/terraform-1.14.8 apply -auto-approve -var-file=terraform.tfvars
```

- [x] Terraform apply succeeds
