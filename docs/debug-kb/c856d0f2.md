# KB Entry: Firebase deploy failed because Cloud Billing API was disabled on the project

## Context
- Command: `npm run release:deploy`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: after fixing the deploy identity IAM, the GitHub Actions release reached Firestore/Functions deploy preparation and then failed later in `firebase deploy`.

## Error Signature
```text
Error: Request to https://cloudbilling.googleapis.com/v1/projects/<project>/billingInfo had HTTP Error: 403, Cloud Billing API has not been used in project before or it is disabled.
```

## Attempts

### Attempt 1
**Date**: 2026-04-02
**Change**: Pulled the full release job log and searched for the actual Firebase CLI failure instead of the truncated `npm run deploy:log` wrapper error.

**Result**: ✅ Confirmed the direct blocker was `cloudbilling.googleapis.com` being disabled on the production project.

### Attempt 2
**Date**: 2026-04-02
**Change**: Enabled `cloudbilling.googleapis.com` on project `courtmaster-v2` and updated Terraform bootstrap required services to include it permanently.

**Result**: ✅ Service state changed to `ENABLED` and the bootstrap layer now codifies the dependency for future environments.

## Root Cause
Firebase CLI checks project billing information during production deploys, but the production project had never enabled the Cloud Billing API and the bootstrap Terraform service list did not include it.

## Fix (final)
```bash
ACCESS_TOKEN=$(gcloud auth application-default print-access-token)
curl -X POST \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  "https://serviceusage.googleapis.com/v1/projects/137312981992/services/cloudbilling.googleapis.com:enable"
```

Then codify it in:
```hcl
infra/terraform/bootstrap/main.tf
```

## Verification
- [x] `https://serviceusage.googleapis.com/v1/projects/137312981992/services/cloudbilling.googleapis.com` returns `ENABLED`
- [x] `infra/terraform/bootstrap/main.tf` includes `cloudbilling.googleapis.com`
- [x] Subsequent release retries no longer fail on `cloudbilling.googleapis.com` disabled
