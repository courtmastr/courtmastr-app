## Root Cause
Compact schedule table used centralized slot-state labels, but full schedule grid used raw participant lookup and defaulted missing slots to `Unknown/TBD`; additionally, BYE classification required finalized status/winner, so round-1 structural byes leaked as schedulable in some flows.

## Fix (final)
```bash
# 1) Centralize slot semantics
# - useMatchSlotState: classify round-1 one-sided slots as BYE
# - keep later-round one-sided slots as TBD

# 2) Enforce schedulability in Match Control schedule state
# - getMatchScheduleTime/getMatchScheduleState return unscheduled for non-schedulable (BYE)

# 3) Reuse shared slot-state in full schedule grid
# - labels from getSlotLabel (BYE/TBD/resolved)
# - scheduled grid excludes non-schedulable matches
```
