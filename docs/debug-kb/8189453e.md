# KB Entry: GitHub OIDC deploy identity lacked actAs on the Compute Engine default service account

## Context
- Command: `npm run release:deploy`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Prior hotfixes had already granted `Service Usage Consumer` and enabled `cloudbilling.googleapis.com`, allowing the release to reach Cloud Functions v2 updates.

## Error Signature
```
Caller is missing permission 'iam.serviceaccounts.actAs' on service account projects/-/serviceAccounts/PROJECT_NUMBER-compute@developer.gserviceaccount.com
```

## Attempts

### Attempt 1
**Date**: 2026-04-02
**Change**: Reviewed the failed GitHub Actions deploy log after the release progressed past previous IAM and billing blockers.

**Result**: ✅ Identified that Firebase Functions v2 was attempting to update functions with the Compute Engine default service account, not just the App Engine default service account.

### Attempt 2
**Date**: 2026-04-02
**Change**: Updated `infra/terraform/deploy/main.tf` to grant `roles/iam.serviceAccountUser` on `${PROJECT_NUMBER}-compute@developer.gserviceaccount.com` to `firebase-deploy@courtmaster-v2.iam.gserviceaccount.com`, then applied the Terraform deploy layer.

**Result**: ✅ Terraform created `google_service_account_iam_member.compute_default_act_as` successfully.

## Root Cause
The deploy identity only had `actAs` on `courtmaster-v2@appspot.gserviceaccount.com`, but Cloud Functions 2nd Gen also required `actAs` on the project’s Compute Engine default service account during function create/update operations.

## Fix (final)
```bash
GOOGLE_APPLICATION_CREDENTIALS=$HOME/.config/gcloud/application_default_credentials.json \
  /Users/ramc/.local/bin/terraform-1.14.8 \
  -chdir=/Users/ramc/Documents/Code/courtmaster-v2/infra/terraform/deploy \
  apply -auto-approve \
  -var-file=/Users/ramc/Documents/Code/courtmaster-v2/infra/terraform/deploy/terraform.tfvars
```

Terraform must include:
```hcl
data "google_project" "current" {
  project_id = var.project_id
}

locals {
  compute_default_service_account_email = "${data.google_project.current.number}-compute@developer.gserviceaccount.com"
}

resource "google_service_account_iam_member" "compute_default_act_as" {
  service_account_id = "projects/${var.project_id}/serviceAccounts/${local.compute_default_service_account_email}"
  role               = "roles/iam.serviceAccountUser"
  member             = "serviceAccount:${google_service_account.firebase_deploy.email}"
}
```
