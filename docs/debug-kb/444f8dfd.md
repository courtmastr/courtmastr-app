# KB Entry: functions build fails when volunteer secret binding uses unavailable runWith API

## Context
- Command: `npm run build:log --prefix functions`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Added durable volunteer secret configuration for local + production.

## Error Signature
```
src/volunteerAccess.ts: TS2339 Property 'runWith' does not exist on firebase-functions v2 namespace.
```

## Attempts

### Attempt 1
**Date**: 2026-03-12
**Change**: Added `functions.runWith({ secrets: [...] })` wrappers around volunteer callables.

**Result**: ❌ Functions TypeScript build failed because this codebase resolves `firebase-functions` to the v2 API surface that does not expose `runWith`.

### Attempt 2
**Date**: 2026-03-12
**Change**: Migrated to `functions.https.onCall({ secrets: [...] }, handler)` for each volunteer callable.

**Result**: ✅ `npm run build --prefix functions` and `npm run build:log --prefix functions` both passed.

## Root Cause
The repo is typed against `firebase-functions` v2 callables, where runtime secret binding must be passed via the `onCall` options object, not via `runWith`.

## Fix (final)
```bash
# Use v2 callable secret options instead of runWith:
functions.https.onCall({ secrets: ['VOLUNTEER_PIN_SECRET'] }, async (request) => { ... })
```

## Verification
- [x] Functions build passes: `npm run build --prefix functions`
- [x] Functions build log passes: `npm run build:log --prefix functions`
- [x] Volunteer callables now declare required secret bindings directly
