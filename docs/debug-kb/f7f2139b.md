# KB Entry: Firestore adapter stringifies numeric bracket foreign keys

## Context
- Command: `npm run test:log -- --run tests/unit/firestoreAdapter.test.ts`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Added a regression test for volunteer manual scoring completion after tracing a live `updateMatch` failure with `Error getting rounds.`

## Error Signature
```
AssertionError: round lookup returns null because stage_id/group_id query values were converted to strings
```

## Attempts

### Attempt 1
**Date**: 2026-03-12
**Change**: Added a failing adapter test that selects `round` documents with `stage_id: 0` and `group_id: 0`.

**Result**: ❌ The query returns `null`, confirming that bracket foreign key filters are being normalized incorrectly.

## Root Cause
`functions/src/storage/firestore-adapter.ts` normalized bracket foreign-key filters such as `stage_id` and `group_id` to strings, but the bracket collections store those IDs as numbers, so `BracketsManager.update.match()` could not load rounds during manual-score completion.

## Fix (final)
```bash
# 1) Update FirestoreStorage.normalizeQueryValue() so bracket foreign keys
#    preserve numeric IDs instead of stringifying them.
# 2) Add a regression test that selects round documents with numeric
#    stage_id/group_id filters.
# 3) Rebuild functions and re-run the volunteer manual fallback flow.
```
