# KB Entry: build:log fails on front desk check-in refs/computed typing in composable and view

## Context
- Command: `npm run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2-frontdesk`
- Recent changes: Added new front desk check-in composable, rapid/bulk components, route, and view.

## Error Signature
```text
TS18049 on bib filter guard + TS2339/TS2740 from using workflow nested ComputedRef properties directly in template bindings.
```

## Attempts

### Attempt 1
**Date**: 2026-02-26  
**Change**: Tightened bib filter type guard to `value != null && value > 0` and promoted nested workflow refs to top-level bindings in the view (`stats`, `urgentItems`, `recentItems`, `bulkRows`, `bulkUndoToken`) so template ref-unwrapping works with strict typing.

**Result**:
✅ `npm run build:log` succeeded.
Artifact:
- `2026-02-26-06-35-06.vue-tsc-b.log`

## Root Cause
Nested `ComputedRef` properties from a plain object return were passed directly to template and TypeScript type-checking treated them as refs rather than unwrapped arrays/objects.

## Fix (final)
```bash
# Code changes
# 1) useFrontDeskCheckInWorkflow.ts
#    - Replace Boolean type guard with explicit null-safe numeric guard.
# 2) FrontDeskCheckInView.vue
#    - Create top-level bindings:
#      const urgentItems = workflow.urgentItems
#      const recentItems = workflow.recentItems
#      const bulkRows = workflow.bulkRows
#      const stats = workflow.stats
#      const bulkUndoToken = workflow.bulkUndoToken
#    - Bind template to those top-level refs.

npm run build:log
```
