# KB Entry: build:log fails when Vue router helper is typed as Promise<void>

## Context
- Command: `npm run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Added merge navigation helpers to player list/profile views while wiring merge UI discoverability.

## Error Signature
```
Type 'Promise<void | NavigationFailure | undefined>' is not assignable to type 'Promise<void>'
```

## Attempts

### Attempt 1
**Date**: 2026-04-02
**Change**: Replaced helper return annotations in `PlayersListView.vue` from `Promise<void>` to `void` and used `void router.push(...)` because the click handlers do not consume the navigation result.

**Result**:
✅ Build succeeded

## Root Cause
Vue Router's `router.push()` resolves to `void | NavigationFailure | undefined`, so annotating fire-and-forget click helpers as `Promise<void>` narrows the real return type and breaks `vue-tsc`.

## Fix (final)
```bash
# In click-only helpers, do not type router.push wrappers as Promise<void>
# Prefer:
const goSomewhere = (): void => {
  void router.push({ name: 'route-name' });
};

npm run build
npm run build:log
```

## Notes
- If a caller genuinely needs to react to navigation completion/failure, do not discard the promise; widen the annotation or `await` it in an async function.
