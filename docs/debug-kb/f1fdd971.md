## Root Cause
Volunteer scorekeeper mode was still sharing the staff scoring list behavior, so the kiosk opened `ManualScoreDialog` instead of the point-by-point scorer and the scoring view still treated volunteers like staff for back-navigation and correction access.

## Fix (final)
```bash
# 1) Add a volunteer-only scoring detail route guarded by the scorekeeper volunteer session.
# 2) Branch MatchListView so volunteer scorekeepers route into the shared scoring interface
#    with the required category query, while staff keeps the manual dialog flow.
# 3) Update ScoringInterfaceView to treat volunteer scorekeepers as a first-class mode:
#    - back target returns to /scoring-kiosk
#    - completed matches allow correction
#    - manual entry is secondary "Manual Fallback"
#    - mobile scoring cards stack with large undo buttons
```
