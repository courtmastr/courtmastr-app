# KB Entry: test:log fails when the Firebase mock omits Timestamp in public snapshot tests

## Context
- Command: `npm run test:log -- --run tests/unit/usePublishSnapshot.standings.test.ts`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2/.worktrees/pool-standings-single-source`
- Recent changes: Added a regression test for public snapshot standings so `/t/:slug` exports the same pool order and match points as smart-bracket standings.

## Error Signature
```
Error: [vitest] No "Timestamp" export is defined on the "@/services/firebase" mock.
```

## Attempts

### Attempt 1
**Date**: 2026-04-19
**Change**: Added a minimal `MockTimestamp` class to the `@/services/firebase` mock and exported it as `Timestamp` so `convertTimestamps()` can detect timestamp-like values during snapshot publishing.

**Result**:
✅ `npm run test:log -- --run tests/unit/usePublishSnapshot.standings.test.ts` advanced past the mock failure.

## Root Cause
The new snapshot regression test mocked Firebase helpers but omitted the `Timestamp` constructor that the snapshot publish path imports through `convertTimestamps()`.

## Fix (final)
```bash
# In the Vitest Firebase mock, export a Timestamp-compatible class
class MockTimestamp {
  constructor(private readonly date: Date) {}
  toDate(): Date {
    return this.date;
  }
}

vi.mock('@/services/firebase', () => ({
  Timestamp: MockTimestamp,
  // ...other mocked exports
}));

npm run test:log -- --run tests/unit/usePublishSnapshot.standings.test.ts
```
