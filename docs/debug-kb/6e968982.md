# KB Entry: build:log fails when a hoisted Vue test import is left untyped

## Context
- Command: `npm run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2/.worktrees/pool-standings-single-source`
- Recent changes: Added a `SmartBracketView` regression test covering `Pre-Elim Snapshot` population after elimination.

## Error Signature
```
TS2347: Untyped function calls may not accept type arguments.
```

## Attempts

### Attempt 1
**Date**: 2026-04-19
**Change**: Replaced the untyped `const { ref } = require('vue')` pattern with a typed `const vue = require('vue') as typeof import('vue')` in the hoisted test runtime and used `vue.ref<T>()` so `vue-tsc` could infer the generic calls correctly.

**Result**:
✅ `npm run build` and `npm run build:log` both passed.

## Root Cause
`require('vue')` inside the hoisted test runtime was inferred as `any`, so the generic `ref<T>()` calls in the new test file were treated as invalid untyped calls during `vue-tsc`.

## Fix (final)
```bash
const runtime = vi.hoisted(() => {
  // eslint-disable-next-line @typescript-eslint/no-require-imports
  const vue = require('vue') as typeof import('vue');
  return {
    categories: vue.ref<Array<Record<string, unknown>>>([]),
  };
});

npm run build
npm run build:log
```
