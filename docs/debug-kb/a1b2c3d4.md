# KB Entry: Participant ID Not Found

## Context
Generating tournament bracket using brackets-manager with FirestoreStorage adapter. Participants are created successfully but brackets-manager throws "Participant id not found in database" when calling manager.create.stage().

## Attempt 4
**Date**: 2026-01-31
**Change**: Modified normalizeReferences in ClientFirestoreStorage to NOT convert 'id' fields to strings
**Details**: Changed condition from `key.endsWith('_id') || key === 'id'` to `key.endsWith('_id') && key !== 'id'`
**Reason**: brackets-manager compares participant.id (number) with seeding values (numbers) using strict equality (===). String '1' !== number 1.
**Result**: 🔄 Testing now...

## Root Cause
brackets-manager's mapParticipantsToDatabase uses strict equality (===) to match participant.id with seeding slot values. FirestoreStorage was converting ALL id fields to strings, but seeding array contains numbers. Comparison '1' === 1 returns false.

## Fix (final)
In src/services/brackets-storage.ts, modify normalizeReferences() to only normalize foreign key references (fields ending in _id), not primary id fields:

```typescript
// Before:
if (key.endsWith('_id') || key === 'id') {
  normalized[key] = String(value);
}

// After:
if (key.endsWith('_id') && key !== 'id') {
  normalized[key] = String(value);
}
```

This preserves participant.id as number for brackets-manager compatibility while still normalizing foreign key references like tournament_id.
