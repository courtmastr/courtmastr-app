# KB Entry: functions build fails when volunteer session payload narrowing leaves numeric fields possibly undefined

## Context
- Command: `npm --prefix functions run build:log`
- Working directory: `/Users/ramc/Documents/Code/courtmaster-v2`
- Recent changes: Added `functions/src/volunteerAccessCore.ts` and `functions/src/volunteerAccess.ts` for volunteer PIN/session support.

## Error Signature
```
src/volunteerAccessCore.ts: Type 'number | undefined' is not assignable to type 'number'
```

## Attempts

### Attempt 1
**Date**: 2026-03-11
**Change**: Ran the Functions TypeScript build immediately after adding the new volunteer access files.

**Result**: ❌ Failed because `assertValidPayload()` validated numeric fields but returned `candidate.pinRevision`, `candidate.issuedAtMs`, and `candidate.expiresAtMs` without converting them to concrete numbers for TypeScript.

### Attempt 2
**Date**: 2026-03-11
**Change**: Normalized the validated numeric fields with `Number(...)` before returning the payload from `assertValidPayload()`.

**Result**: ✅ `npm --prefix functions run build:log` passed.

## Root Cause
Runtime guards proved the payload numbers were present, but the function still returned `Partial<VolunteerSessionPayload>` fields directly, leaving TypeScript with `number | undefined`.

## Fix (final)
```bash
# After validating the payload fields, convert the optional values to concrete numbers.
npm --prefix functions run build:log
```
