## Root Cause

The regression test mounted `ScoringQrDialog` with `modelValue: true`, but the component only generates the QR code when the dialog transitions from closed to open, so the watcher never fired during the initial mount.

## Fix (final)

1. Mount the dialog test with `modelValue: false`.
2. Call `wrapper.setProps({ modelValue: true })` before asserting QR generation behavior.
3. Keep the test focused on the route resolution and QR payload rather than assuming eager mount-time generation.
